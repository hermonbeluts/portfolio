Clicker Data Dashboard

Building Interactive Learning Analytics Dashboards | Spring 2024

Project Overview

The goal of this project was to develop a learning analytics dashboard based on clicker data from previous students of a CS course at Cornell University. We were tasked with using RShiny to develop dashboards given aggregated clicker records and quiz data. Using RShiny, we created two dashboards: an instructor dashboard, which offers an overview of the class characteristics, engagement, and performance, and a student dashboard, which offers a specific student an overview of their engagement and performance (and how it compares to others).

Datasets

We worked with two datasets:

  • Experience Dataset: The experience dataset contains one record per student who completed the CS course between 2016-2018. There are two sources to this dataset: Faculty Center and a Skills Survey (administered via the Blackboard LMS) where students self-reported their skill level for various skills the first week of class. This data has been de-identified. Name, NETID, emplid, and major have all been removed and replaced with a unique numeric identifier. Note that not all students completed the skills survey, they will have null values for the survey result fields. The data contains several attributes such as prog (whether a student was GRAD or UGRAD), database_score, sql_score, programming_score (self-reported experience levels), and more.
  • Quiz Dataset: The quiz dataset contains one record per student per class session held where iClickers were used. Sources used in the creation of this data set include: iClicker session xml files, Blackboard gradebook (for quiz scores), and the Blackboard class schedule (used to map iClicker session to related quiz scores). Note that in some cases there are multiple iClicker sessions / lectures associated with a single quiz. This dataset may be joined to the experience dataset by the student_key field. The data contains several attributes such as attended (measuring student attendance), correct_q_clicker (the number of correct clicker questions answered by a student), and more.

Process

Our work involved the following steps:

  • Planning: Determined the target audience and their needs, what the goals of our dashboards were, and how the visualizations would support our goal (for both the student and teacher dashboards). We proceeded to sketch each dashboard, each containing at least 4 data visualizations.
  • Wireframe Implementation: Generated the dashboard layout given a very basic wireframe
  • Data Pre-processing: Cleaned and merged datasets to prepare them for visualization.
  • Visualization: Created data visualizations for both dashboards to display key metrics.

Dashboards

  • Instructor Dashboard: Displays metrics such as: the total number of students, the average attendance, the distribution of students by program, the distribution of experience levels in different categories, the average quiz score per quiz separated by program, the average temperature of students per session, and average attendance per session.
  • Student Dashboard: Displays metrics such as: their quiz average, the number of classes missed, their clicker progress bar, their temperature every session, and two interpretive visualizations for quiz scores.
